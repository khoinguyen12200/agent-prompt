# Step 3: Build

## Purpose
Implement the planned solution. This is where ideas turn into code.

## What the Agent Must Do
1. **Execute the plan.** Work through subtasks in a logical order.
2. **Write clean, maintainable code.** Follow the project's coding style and patterns.
3. **Use tools effectively.** Use `WriteFile`, `StrReplaceFile`, `Shell`, and other tools to make real changes.
4. **Stay focused.** Complete one subtask at a time. Update the todo list if used.
5. **Handle blockers.** If the plan needs to change mid-flight, pause, reassess, and update the plan if necessary.

## Rules
- **Make minimal changes.** Only modify what is required to achieve the goal.
- **Follow existing conventions.** Match the project's naming, formatting, and architectural patterns.
- **Do not break existing logic.** Especially in tests—unless the test itself is the target of the fix.
- **Write code, don't just describe it.** Every change must be made via tools, not just explained in chat.
- **Iterate.** If something doesn't work, fix it before moving on.
- **Keep the user informed.** Provide brief updates on progress for long-running tasks.
