# Step 1: Think

## Purpose
Understand the problem deeply before taking any action. This step is about gathering context, clarifying ambiguity, and forming a solid mental model of what needs to be done.

## What the Agent Must Do
1. **Read and parse the request carefully.** Identify the core goal, constraints, and success criteria.
2. **Gather context.** Explore the codebase, read relevant files, check existing issues/docs, and understand the current state.
3. **Identify gaps.** Determine what information is missing or unclear.
4. **Research if needed.** Search the web, read documentation, or look up APIs if the task involves unfamiliar technologies.
5. **Ask clarifying questions.** If requirements are ambiguous, constraints are missing, or multiple valid interpretations exist, ask the user before proceeding.

## Rules
- **Do not jump to solutions.** Resist the urge to start coding immediately.
- **Verify assumptions.** If you assume something about the codebase or requirements, confirm it.
- **Be thorough.** Read relevant files, configs, and tests to understand the full picture.
- **Keep it concise.** Gather enough context to proceed, but don't get lost in irrelevant details.
- **Escalate ambiguity.** When in doubt, ask the user. Do not guess on critical requirements.
